# v0.5 — Lς (stale-lock guard) joins the canon

Released: 2026-05-28

A new Polish-Phase5 lesson, **Lς (stale `.git/index.lock` cascade
failure)**, is promoted to scaffold-level canon. v0.5 ships the
substrate-level guard so every capability portrait inheriting from
this template gets stale-lock protection for free in its
`commit_template.sh`.

## What's new

### `scripts/commit_template.sh` (Lς guard added)

Before any staging or commit, the script now:

```bash
if [ -f .git/index.lock ]; then
  echo "=== Lς: removing stale .git/index.lock ==="
  rm -f .git/index.lock
fi
git status >/dev/null 2>&1 || { echo "git status failed; abort"; exit 1; }
```

This closes the ghost-shipping failure mode described in Lς (see
`docs/commit-script-pattern.md`).

### `docs/commit-script-pattern.md` (Lς section added)

Full Symptom / Cause / Fix / Generalizing / Provenance write-up of
the stale-lock cascade.

## Why this lesson now

Surfaced during DMOI v0.6 ship on 2026-05-28. A prior session left
`.git/index.lock` in place. The user's pasted heredoc Mac ship
sequence emitted `fatal: ... index.lock: File exists` on the
`git add`, `git commit`, and `git rm` lines (easy to miss when
scrolling output), but `git tag`, `git push --tags`, and
`gh release create` are not index-mutating and proceeded
successfully. Result: a v0.6 tag and GitHub Release were created
pointing at the v0.5 cleanup commit `8a97d11`, not at any new
v0.6 commit (because none existed). Recovery required
`gh release delete v0.6 --cleanup-tag --yes` and re-running the
ship sequence from a clean state.

The Lς guard makes this failure mode self-healing: the script
clears any stale lock at the top, so subsequent operations succeed.

## What's not in v0.5 (honest scope)

- The guard only protects the `commit_template.sh`-derived ship
  scripts. Ad-hoc heredoc paste sequences typed directly into a
  shell are still vulnerable. The doc recommends starting any such
  paste with `rm -f .git/index.lock` as the first line.
- No code-level changes outside `scripts/commit_template.sh` and
  the two docs. Existing capability portraits that have already
  customized their `commit_dayN.sh` will need to backport the
  6-line Lς block manually (or re-copy from the v0.5 template).
- The scanner globs in `scripts/check_english_only.py` are
  unchanged from v0.4. Lς is a workflow lesson, not a content
  lesson — no new file patterns need english-only enforcement.
