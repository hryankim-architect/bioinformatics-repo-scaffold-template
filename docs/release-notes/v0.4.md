## What's new

### `src/bioscaffold/saturation.py` (new)

Reusable LÏƒ primitive. Exports:

- **`check_saturation(values, *, metric, threshold, near_threshold,
  candidate_causes, direction)`**, returns a `SaturationReport`.
- **`SaturationReport`**, frozen dataclass with `is_saturated`,
  `is_near_saturated`, descriptive stats (`max_value`, `min_value`,
  `mean_value`, `n_values`), and `audit_section()` that renders the
  appropriate Markdown section (Saturation finding / Near-saturation
  warning / Scope).

Supports both `direction="max"` (higher = better; AUROC, accuracy, F1)
and `direction="min"` (lower = better; MAE, MSE, loss).

Adoption pattern in a capability portrait's baseline driver:

```python
from bioscaffold.saturation import check_saturation

# After computing per-fold metrics
auc_means = [agg[(fs, m)]["auc_mean"] for fs, m in keys]
report = check_saturation(
    auc_means,
    metric="AUROC",
    candidate_causes=(
        "Was the label derived from one of the input modalities?",
        "Are the two classes biologically very distinct cell types?",
    ),
)
audit_md.write_text(header + report.audit_section() + reproduce_section)
```

### `src/bioscaffold/vocab.py` (new)

Reusable LÏ‰ primitive. Exports:

- **`defensive_filter(df, *, primary_col, allowed_values, fallback_col,
  fallback_map, raise_on_empty, label)`**, returns a
  `DefensiveFilterResult` with `matched`, `normalized_column`, and
  diagnostic counts (`unmatched_count`, `primary_hit_count`,
  `fallback_hit_count`, `unique_values_seen`).
- **`VocabMismatchError`**, raised when no rows match and
  `raise_on_empty=True` (the default). The message includes the columns
  checked + a summary of the upstream values actually seen, so the
  diagnostic points at the likely cause (vocabulary drift) rather than
  surfacing as a cryptic downstream `KeyError`.

Adoption pattern for cohort selection against TCGA / GDC / Xena:

```python
from bioscaffold.vocab import defensive_filter

PAM50_LONG_TO_SHORT = {"Luminal A": "LumA", "Luminal B": "LumB", ...}
result = defensive_filter(
    clinical_df,
    primary_col="PAM50Call_RNAseq",
    fallback_col="PAM50_mRNA_nature2012",
    fallback_map=PAM50_LONG_TO_SHORT,
    allowed_values={"LumA", "LumB"},
    label="patients",
)
print(f"cohort: {len(result.matched)} ({result.primary_hit_count} primary, "
      f"{result.fallback_hit_count} fallback, {result.unmatched_count} unmatched)")
```

## Migration from v0.3 to v0.4

Pure additions; no breaking changes. To adopt the helpers in a downstream
capability portrait:

1. Pull v0.4 (`git pull origin main`).
2. Import the helpers where you compute baseline metrics or filter
   against upstream vocabularies.
3. Replace ad-hoc saturation detection / vocabulary filtering with the
   substrate calls.
