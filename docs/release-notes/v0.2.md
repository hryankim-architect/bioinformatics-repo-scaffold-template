# v0.2, commit script template

Released: 2026-05-27

The first substrate-level minor bump after v0.1 (the initial scaffold).
Promotes an engineering lesson from the DMOI POC Week-1 sprint
(`dmoi-brca-poc` Days 1–5B, six CI-green commits) into a scaffold-level
pattern every future repo created from this scaffold inherits.

## What's new

### `scripts/commit_template.sh` (new)

A copy-and-customize template for any `commit_dayN.sh` helper. Bakes
in a pre-push pattern:

- **Lχ, heredoc commit messages.** Uses `git commit -F-` with a
  heredoc body, never inline `-m "..."`. This sidesteps `zsh`
  `BANG_HIST` mangling any `!` or `$!,` inside the message.

The template is tolerant of missing dev deps locally (`ruff`, `pytest`):
if they're not installed it prints a notice and proceeds, CI remains
the canonical gate.

### `docs/commit-script-pattern.md` (new)

The Lχ lesson is documented in `Symptom / Cause / Fix / Generalizing` form,
plus adoption instructions for new day-N commits. Becomes the reference
that downstream repos link to from their own `docs/` when
they describe their commit cadence.

## Field-tested evidence

DMOI POC Week-1 sprint (2026-05-26 → 2026-05-27):

- 6 commits across Days 1–5B.
- Zero `zsh BANG_HIST` hits after applying the heredoc pattern (it had
  surfaced 2× during early Day-2 development before the lesson landed).
- All 6 commits CI-green on first push.

The Lχ lesson came from the dmoi-brca-poc sprint but the pattern
generalizes to every repo in this scaffold family; it's now scaffold-level.

## Migration for capability repos inheriting from this scaffold

If you already have `commit_*.sh` helpers in a downstream capability
repo, the recommended migration is:

1. Pull the v0.2 changes (`commit_template.sh`,
   `docs/commit-script-pattern.md`).
2. Either replace your existing `commit_*.sh` with `commit_template.sh`
   + your day-specific staging list, OR add the heredoc
   commit-message pattern to your existing helper.

The v0.1 scaffold remains source-compatible. v0.2 only adds, no
breaking changes.

## What's not in v0.2 (scope)

- A pre-push hook auto-installer (e.g. `.git/hooks/pre-push`).
  Intentionally kept manual; pre-push gates run inside `commit_*.sh`
  by convention. Auto-install can be a v0.3 candidate if the manual
  pattern proves too easy to forget.
- A general lessons index. Lessons are referenced ad
  hoc in `docs/commit-script-pattern.md`; a unified index across all
  ~132 captured lessons remains internal-only for now.
