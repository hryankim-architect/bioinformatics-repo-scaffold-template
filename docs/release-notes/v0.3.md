# v0.3, lesson canon (Lψ + Lω + Lσ)

Released: 2026-05-27

Docs-only minor bump promoting three more engineering lessons from the
DMOI POC Week-1 sprint to scaffold-level reference. v0.2 shipped two
patterns embedded directly in code (Lχ heredoc commit messages, Lτ
pre-push CJK gate). v0.3 ships three patterns whose substrate expression
is documentation-first; the reusable code helpers come in v0.4.

## What's new

### `docs/lessons/polish-phase5.md` (new)

The canonical lesson list, scoped to the lessons that have
been promoted to scaffold level (vs. the ~132 internal lessons captured
during substrate development). Each lesson is documented in the standard
`Symptom / Cause / Fix / Generalizing` form, plus a brief promotion-criteria
note at the end.

Lessons indexed in v0.3:

- **Lχ**, heredoc commit messages (no inline `-m`), v0.2 code, documented
  in `docs/commit-script-pattern.md`.
- **Lτ**, pre-push CJK gate, v0.2 code + scanner expansion.
- **Lψ** (new in v0.3 docs), dev-tool tolerance in commit scripts: probe
  for `ruff` / `pytest` before running, skip with notice if not installed.
  Already embedded in `scripts/commit_template.sh` from v0.2; v0.3 documents
  the *why*.
- **Lω** (new), defensive vocabulary handling against upstream public
  datasets: primary + fallback columns, `raise ValueError` on empty
  filter results, log unmatched-row counts.
- **Lσ** (new), saturation as substrate primitive: detect AUROC ≥ 0.99
  in baseline outputs and emit a structurally-different "Saturation
  finding" section in the audit MD. Forces honest re-scope conversation
  instead of silently shipping the misleading result.

### Promotion criteria documented

The lesson file ends with a brief section on when an internal lesson
graduates to scaffold-level promotion:

1. Re-encountered in **at least two** project sprints.
2. Fix has a substrate-level expression (script, doc, helper).
3. Fix has been field-tested on at least one CI-green commit cycle.

Lψ, Lω, Lσ all cleared this bar during DMOI POC Week-1.

## What's not in v0.3 (scope)

- Reusable Python helpers for Lω and Lσ. The patterns are codified as
  documentation only. A planned v0.4 will ship:
  - `src/scaffold/saturation.py` with `check_saturation()` returning
    a `SaturationReport` dataclass.
  - Possibly `src/scaffold/vocab.py` with a `defensive_filter()` helper
    that surfaces unmatched-row counts.
- An automated lesson-promotion workflow (e.g. a GitHub Action that
  flags internal lessons due for promotion). Manual promotion-criteria
  check remains the default.

## Migration

Pure additions, pull the new docs file and the v0.3 release notes. No
behavior changes from v0.2.

```bash
git pull origin main
ls docs/lessons/polish-phase5.md   # new in v0.3
```
